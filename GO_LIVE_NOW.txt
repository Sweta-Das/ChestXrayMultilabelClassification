PYTORCH GRADCAM SETUP
═════════════════════

- ONNX is used for inference and the occlusion fallback by default.
- To enable PyTorch Grad-CAM from `models/best_medfusionnet.pth`, create a dedicated MedFusionNet backend environment from `requirements.medfusion.txt`.
- On macOS, use Python 3.10 with `torch==2.5.1` and `torchvision==0.20.1`, which are a matched pair for the MedFusionNet wrapper.
- `start.sh` will create `.venv-medfusion` automatically if it does not exist, or you can create it manually with:

  `python3.10 -m venv .venv-medfusion`

- If the MedFusionNet env is not set up, the backend still runs and falls back to ONNX occlusion, but you will not get PyTorch Grad-CAM.

DOCKER / LINUX ROUTE
════════════════════

If the native Mac MedFusionNet environment keeps failing, run the backend in Docker:

1. Build and start the Linux backend:

   `docker compose up --build backend`

2. Keep the frontend on your Mac in a second terminal:

   `cd frontend && npm run dev`

3. Point the frontend at the Docker backend if needed:

   `export NEXT_PUBLIC_API_URL=http://localhost:8000`

4. Open the app in your browser:

   `http://localhost:3000`

Notes:
- The container uses `pdflatex` in PATH. If report generation ever complains about missing LaTeX packages, install the extra TeX packages in `Dockerfile.backend` or switch the report flow to the ReportLab fallback.

IF SOMETHING GOES WRONG
════════════════════════════════════════════════════════════════════════════════

Backend won't start:
  Problem: Port 8000 in use
  Solution: pkill -f "python3 api.py"
           Then try again: python3 api.py

Frontend won't start:
  Problem: Port 3000 in use
  Solution: pkill -f "node"
           Then try again: npm run dev

Nothing shows in browser:
  Problem: Frontend not running
  Solution: Check Terminal 2 shows "✓ Ready in X.Xs"
           Check browser console (F12) for errors

API not responding:
  Problem: Backend not running
  Solution: Check Terminal 1 shows "Uvicorn running on..."
           Try curl http://localhost:8000
