SYSTEM OVERVIEW
================================================================================
This project is a modern, production-ready AI system for chest X-ray analysis and medical report generation. It features:

- Deep learning model (CTransCNN, ONNX) for multi-label disease classification
- Retrieval-Augmented Generation (RAG) pipeline for clinical context and impression text (OpenAI GPT-4o-mini)
- GradCAM explainability for model predictions
- LaTeX-based PDF report generation
- Professional Next.js (TypeScript) frontend
- FastAPI backend (RESTful, async)

--------------------------------------------------------------------------------
DATA FLOW
--------------------------------------------------------------------------------
1. User uploads chest X-ray via frontend (drag-drop)
2. Frontend calls POST /api/upload (FastAPI)
3. Backend saves image, returns session_id
4. Frontend calls POST /api/inference with session_id
5. Backend runs ONNX model, returns disease probabilities
6. Frontend displays results, allows GradCAM heatmap generation (POST /api/gradcam)
7. Frontend calls POST /api/report to generate medical report
8. Backend uses RAG pipeline (retriever, OpenAI, prompt templates) for impression
9. Backend renders LaTeX report, compiles PDF
10. Frontend allows PDF download (GET /api/download/{session_id})

--------------------------------------------------------------------------------
API ENDPOINTS (FastAPI)
--------------------------------------------------------------------------------
POST   /api/upload        Upload X-ray image
POST   /api/inference     Run model prediction
POST   /api/gradcam       Generate GradCAM heatmaps
POST   /api/report        Generate medical report (PDF)
GET    /api/download/{session_id}  Download PDF report
GET    /api/session/{session_id}   Get session data
DELETE /api/session/{session_id}   Delete session

TECHNOLOGY STACK

--------------------------------------------------------------------------------
DESIGN PRINCIPLES
--------------------------------------------------------------------------------
- Separation of concerns: clear API boundary between backend and frontend
- Modular: each function (inference, explainability, reporting, retrieval) is isolated
- Scalable: RESTful API, async backend, production-ready frontend
- Explainable: GradCAM overlays, RAG-generated clinical context
- Professional: LaTeX PDF reports, responsive UI, dark mode
